# Claude Code Configuration Reference

| Feature | Possible Locations | Loading Behavior / Priority | Question It Answers | Example .md (in our API project) |
|---|---|---|---|---|
| **CLAUDE.md** | Managed → project (walked up from cwd) → local → user → child-dir | All additive, no hard precedence between project/user. Parent-dir files load at session start; child-dir ones load on demand | "What should Claude always know about this project, regardless of what I'm doing?" | `.claude/CLAUDE.md`: "This is an Express + TypeScript API. Use pnpm, not npm. All endpoints need Zod validation." |
| **Rules** | `.claude/rules/*.md` (project) → `~/.claude/rules/` (user) → plugins | Frontmatter rules load at session start; path-scoped rules load only when a matching file is opened. Additive, not enforced | "What extra instructions apply only when I touch a specific file/folder?" | `.claude/rules/migrations.md`: "When editing `/db/migrations`, never modify an existing file — always create a new one." |
| **settings.json** | Managed → CLI flags → `.claude/settings.local.json` → `.claude/settings.json` → `~/.claude/settings.json` | The one mechanism with true key-by-key precedence: arrays combine, scalars use most-specific value | "What is Claude allowed/forbidden to do, and what are session defaults (model, env)?" | `.claude/settings.json`: `{"permissions": {"deny": ["Bash(rm -rf *)"]}, "model": "claude-sonnet-5"}` |
| **Skills** | `.claude/skills/<name>/SKILL.md` (project) → `~/.claude/skills/` (user) → plugins | Load on demand — Claude sees descriptions at startup, pulls full content when relevant, or you invoke with `/name` | "How do I package a full repeatable procedure Claude can run whenever it's relevant?" | `.claude/skills/new-endpoint/SKILL.md`: "1. Create route file 2. Add Zod schema 3. Register in router 4. Write test" |
| **Commands (legacy)** | `.claude/commands/*.md` (project) → `~/.claude/commands/` (user) → plugins | Same mechanism as skills now; a command and skill sharing a name → skill wins | "How do I define a quick `/slash` shortcut?" (legacy — use Skills instead) | `.claude/commands/deploy.md`: "Run `npm run build`, then deploy to staging." |
| **Subagents** | `.claude/agents/*.md` (project) → `~/.claude/agents/` (user) → plugins → SDK `agents` option | Runs in a fresh, isolated context. Project definitions beat user ones on name collision | "Who should handle this in a separate context so it doesn't clutter my main conversation?" | `.claude/agents/code-reviewer.md`: "You are a strict senior reviewer. Flag security issues, unhandled promise rejections..." |
| **Hooks** | `settings.json` (any scope) → plugins → SDK `hooks` option | All fire, side by side, for the same event — no override, most-restrictive result wins | "What should automatically run before/after an action?" | In `settings.json`: after every `Edit` → run `npm run lint -- --fix` |
| **MCP servers** | Managed allowlist → `.mcp.json` (project) → `~/.claude.json` (user/local) → plugins → SDK option → claude.ai connectors | All configured servers connect; scopes describe where config lives, not override order | "What external tools/data can Claude reach — GitHub, Jira, a database?" | `.mcp.json`: connects a GitHub MCP server so Claude can read/comment on PRs |
| **Plugins** | `~/.claude/plugins/` (marketplace) → SDK `plugins` option | Just a bundle — everything inside (skills/agents/hooks/MCP) behaves like its native type | "How do we ship a whole toolkit (skills+agents+hooks+MCP) to the whole team at once?" | Installed plugin `acme-toolkit` — bundles your company's shared skills/agents |
| **Output styles** | `.claude/output-styles/` (project) → `~/.claude/output-styles/` (user) | One active style at a time; project beats user on name collision | "What tone/persona should responses take?" | `.claude/output-styles/terse-senior-eng.md`: "No preamble. Assume expert audience. Answer in as few words as possible." |
| **Workflows** | `.claude/workflows/*.js` (project) → `~/.claude/workflows/` (user) | Project workflow beats a same-named personal one | "How do I automate a fixed multi-step pipeline, not just a single ask?" | `.claude/workflows/release.js`: automates version bump → changelog → tag → publish |
| **Auto memory** | `~/.claude/projects/<project>/memory/MEMORY.md` | First 200 lines load at session start automatically; loaded regardless of settingSources | "What has Claude learned about this project across past sessions, unprompted?" | Auto-written: "User prefers pnpm. Tests live in `/tests`, not `/src/__tests__`." |
| **Subagent memory** | `.claude/agent-memory/` (project) or `~/.claude/agent-memory/` (user), per `memory:` frontmatter | Separate from main-session auto memory | "What has one specific subagent learned from repeated use on this repo?" | `.claude/agent-memory/code-reviewer/MEMORY.md`: "This team flags nested ternaries every time — keep doing that." |
| **~/.claude.json** | Home dir | Always read, regardless of settingSources | "What machine-level trust/state persists outside any single project?" | Not a `.md` — stores that you've trusted this repo folder + your personal GitHub MCP token |
| **System prompt** | SDK `systemPrompt` option only (code, not a file) | Injected directly by the developer building on the SDK | "If I'm building my own app on top of Claude Code, how do I set its base identity in code?" | Not a `.md` — `systemPrompt: "You are CodeBot, Acme Corp's internal coding assistant."` |
